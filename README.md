Este projeto consiste no desenvolvimento de um sistema para gerenciamento de livros, permitindo o cadastro, consulta, atualização e exclusão de gêneros, autores e livros.
A aplicação foi desenvolvida utilizando uma arquitetura Full Stack, composta por uma API REST em .NET (C#) no backend e uma Single Page Application (SPA) no frontend, responsável pela interação com o usuário.
O objetivo do projeto é demonstrar boas práticas de desenvolvimento, organização de código, modelagem correta de domínio e aplicação de conceitos modernos de desenvolvimento de software.

Este projeto é composto por dois componentes principais: uma API REST desenvolvida em C# utilizando .NET (backend) e uma aplicação SPA desenvolvida em React (frontend). A seguir estão as instruções para configurar e executar o projeto localmente utilizando o Visual Studio, bem como orientações para publicação utilizando o IIS.
Para executar o projeto, é necessário possuir alguns pré-requisitos instalados na máquina. Recomenda-se utilizar o Visual Studio (versão 2022 ou superior) com o workload ASP.NET and Web Development instalado. Também é necessário ter o .NET SDK versão 8.0 ou superior, o Node.js em versão LTS para execução do frontend em React, além de um banco de dados configurado (como SQL Server, MySQL ou PostgreSQL). Para publicação local do sistema, também é necessário ter o IIS habilitado no Windows.
O primeiro passo é realizar o download do projeto a partir do repositório no GitHub. Para isso, utilize o comando git clone apontando para o endereço do repositório e, em seguida, acesse a pasta do projeto em sua máquina.
Após clonar o repositório, abra a solução do backend no Visual Studio através do arquivo .sln. Com o projeto aberto, localize o arquivo appsettings.Development.json e configure a string de conexão com o banco de dados, ajustando servidor, nome do banco e credenciais de acordo com o ambiente local.
Em seguida, é necessário criar as tabelas no banco de dados. Para isso, utilize o Package Manager Console do Visual Studio e execute o comando Update-Database. Esse processo aplicará automaticamente as migrations configuradas no projeto e criará a estrutura necessária no banco de dados.
Depois de configurado o banco, a API pode ser executada diretamente pelo Visual Studio pressionando F5 ou Ctrl + F5. Ao iniciar a aplicação, a API será disponibilizada localmente e a documentação interativa poderá ser acessada através do Swagger, normalmente no endereço https://localhost:5001/swagger.
Com o backend funcionando, o próximo passo é executar o frontend. Abra um terminal na pasta frontend do projeto e execute o comando npm install para instalar todas as dependências necessárias da aplicação React. Após a instalação, verifique no arquivo responsável pela configuração das chamadas HTTP (geralmente localizado em src/services ou src/api) se a URL base da API está configurada corretamente apontando para o endereço da API local. Em seguida, execute o comando npm start para iniciar a aplicação. O frontend será iniciado normalmente no endereço http://localhost:3000, permitindo a interação com os dados de autores, gêneros e livros por meio da interface web.
Caso seja necessário realizar a publicação da aplicação utilizando o IIS, o primeiro passo é publicar o backend. No Visual Studio, clique com o botão direito no projeto da API e selecione a opção Publish. Escolha o método de publicação em Folder e defina um diretório de saída, por exemplo C:\inetpub\BookManagementAPI. Após concluir a publicação, abra o gerenciador do IIS e crie um novo site apontando para essa pasta. Durante a configuração do site, defina um nome para o site, informe o caminho físico da pasta publicada e configure a porta desejada para acesso.
Também é importante configurar corretamente o Application Pool utilizado pelo site. No IIS, selecione o Application Pool correspondente e configure a opção .NET CLR Version como No Managed Code, pois aplicações .NET modernas utilizam o runtime do próprio ASP.NET Core. Caso a API não funcione corretamente no IIS, pode ser necessário instalar o ASP.NET Core Hosting Bundle, disponível no site oficial da Microsoft. Após a instalação, recomenda-se reiniciar o IIS utilizando o comando iisreset.
Para publicar o frontend, navegue até a pasta do projeto React e execute o comando npm run build. Esse comando irá gerar uma versão otimizada da aplicação dentro da pasta build. Em seguida, copie o conteúdo dessa pasta para um diretório dentro do IIS, por exemplo C:\inetpub\BookManagementFrontend. Depois disso, crie um novo site no IIS apontando para essa pasta e configure uma porta de acesso, como por exemplo a porta 3000.
Após a configuração completa, a API poderá ser acessada diretamente pelo navegador ou por ferramentas de teste através do endpoint configurado no IIS, enquanto o frontend permitirá que o usuário interaja com o sistema através da interface web. É importante garantir que o CORS esteja habilitado na API, permitindo que o frontend realize requisições corretamente.
Seguindo esses passos, será possível executar o projeto tanto em ambiente de desenvolvimento utilizando o Visual Studio quanto em ambiente publicado através do IIS.
